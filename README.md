# 💣 Minesweeper - A C++ & Flutter Learning Journey

这是一个经典的扫雷游戏，但它的背后有一个特别的故事：它的核心逻辑由高性能的 C++ 编写，而它的现代化用户界面则由 Flutter 构建。这个项目是我从底层系统编程到现代化 App 开发的一次完整的学习探索。

## 📖 关于本项目：我的学习之旅

这个项目的初衷纯粹是为了**学习**。我希望通过一个不大不小的项目，将理论知识付诸实践，并挑战一些从未接触过的领域。

整个旅程分为两个核心阶段：

### 1. **C++ 核心引擎的构建**
一开始，我使用 C++ 来实现扫雷的全部核心逻辑。这让我深入实践了：
*   **面向对象编程 (OOP)**：将每一个格子抽象成 `block` 类。
*   **内存管理与数据结构**：使用 `std::vector` 来构建和管理整个棋盘状态。
*   **算法设计**：实现了经典的“泛洪填充”递归算法，用于自动展开空白区域。
*   **项目解耦**：将核心逻辑从 `main` 函数中剥离，封装进一个独立的 `Game` 类，为未来的跨平台调用打下基础。

### 2. **向现代化 UI 的跨越**
当 C++ 核心完成后，我面临了一个新的挑战：如何为它穿上一件漂亮的“外衣”？我不想止步于控制台，而是希望构建一个真正的、可以交互的图形化应用。

*   **技术选型**：我选择了 **Flutter**，因为它承诺了漂亮的 UI 和强大的跨平台能力，尤其是 Material Design 的设计风格深得我心。
*   **架构分离**：我学习了如何将 C++ 代码编译成一个**动态链接库 (.dll)**，将它作为“后端引擎”。而 Flutter 则作为“前端界面”，两者完全分离。
*   **学习 FFI**：整个项目的技术核心是 **C/C++ FFI (Foreign Function Interface)**。我从零开始，学习了如何使用 `dart:ffi` 和 `ffigen` 工具，在 Dart 和 C++ 之间建立一座坚固的桥梁，实现了两者之间高效、安全的数据通信。

最终，我成功地将两个完全不同的技术栈融合在了一起，完成了一个从底层逻辑到上层表现的完整闭环。

## ✨ 功能特性 (Features)

*   **经典扫雷玩法**：左键揭开，右键插旗。
*   **响应式布局**：游戏界面能够自动缩放，适应不同大小的窗口。
*   **C++ 核心驱动**：所有游戏逻辑均由高性能的 C++ 动态库处理。
*   **清晰的反馈**：包含游戏胜利和失败的弹窗提示。
*   **便捷的重玩机制**：可以随时通过刷新按钮或在游戏结束后重新开始。

## 🛠️ 技术栈 (Tech Stack)

*   **后端核心 (Core Engine)**
    *   C++ (C++17/20)
    *   CMake (用于构建动态库)
*   **前端界面 (GUI)**
    *   Dart
    *   Flutter (Material Design)
*   **桥梁技术 (Bridge)**
    *   C FFI (`dart:ffi`, `ffigen`)

## 🚀 如何运行 (Getting Started)
